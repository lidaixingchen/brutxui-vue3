[index.md](../apps/docs/index.md) 中的技术栈版本与 [AGENTS.md](../AGENTS.md) 不一致：

| 项目 | 首页显示 | 实际版本（AGENTS.md） |
|------|----------|----------------------|
| Tailwind CSS | 3.4+ | 4.3+ |
| Vite | 6+ | 8+ |
| Vitest | 3+ | 4+ |

**建议**：统一为实际版本，或改为不写具体版本号以减少维护负担。

## 三、组件文档规范

已创建 [COMPONENT_DOC_TEMPLATE.md](./COMPONENT_DOC_TEMPLATE.md) 作为标准模板。

### 模板摘要

#### 章节顺序规范

| 顺序 | 章节 | 是否必须 | 说明 |
| ---- | ---- | -------- | ---- |
| 1 | 预览 | ✅ | 必须包含 `<ComponentPreview>` |
| 2 | 安装 | ✅ | 使用 `<InstallationTabs>` 组件 |
| 3 | 用法 | ✅ | 至少一个完整示例 |
| 4 | 变体 | 按需 | 有变体时必须 |
| 5 | 尺寸 | 按需 | 有尺寸时必须 |
| 6 | 子组件 | 复合组件必须 | 列出所有子组件 |
| 7 | 数据类型 | 按需 | 有复杂 TypeScript 类型时必须 |
| 8 | 组合式函数 | 按需 | 提供 composable 时必须 |
| 9 | 程序化控制 | 按需 | 通过 defineExpose 暴露 API 时必须 |
| 10 | Props | ✅ | 统一 4 列格式 |
| 11 | 事件 | 按需 | 有自定义事件时必须 |
| 12 | 插槽 | 按需 | 有插槽时必须 |
| 13 | 可访问性 | ✅ | 统一使用此名称 |
| 14 | 样式定制 | 按需 | 支持 CSS 变量自定义时必须 |
| 15 | 常见问题 | 推荐 | 复杂组件建议添加 |

#### 表格格式规范

**Props 表格**（4 列，`class` 属性放最后）：

```markdown
| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
```

**事件表格**（3 列，载荷格式：简单类型直接写，复合载荷用元组带参数名）：

```markdown
| 事件 | 载荷 | 说明 |
|------|------|------|
| `click` | `MouseEvent` | 点击时触发 |
| `sort` | `[column: string, direction: 'asc' \| 'desc']` | 排序变化时触发 |
```

**插槽表格**（3 列，无作用域时用 `—`）：

```markdown
| 插槽 | 作用域 | 说明 |
|------|--------|------|
```

#### 组合式函数与程序化控制

当组件同时有 composable 和 defineExpose 时：

- **组合式函数**：侧重"独立使用"场景
- **程序化控制**：侧重"通过 ref 调用组件实例"场景

#### 可访问性章节

统一使用 `## 可访问性` 作为章节名，不要使用"无障碍"或"Accessibility"。内容按以下顺序组织：

1. 键盘操作
2. ARIA 属性
3. 焦点管理

#### 样式定制章节

支持 CSS 变量自定义的组件必须包含此章节：

```markdown
## 样式定制

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `--brutal-{component}-duration` | `200ms` | 动画时长 |
```

### 抽查发现的格式问题

| 组件 | 问题 | 模板规定 |
|------|------|----------|
| DataTable | Props 表格 3 列，缺少"说明"列 | 应为 4 列：属性/类型/默认值/说明 |
| DataTable | 插槽表格列名用英文 `Slot` 和 `用途` | 应为中文：插槽/作用域/说明 |
| DataTable | 缺少"可访问性"章节 | 必须包含 |
| KanbanBoard | 事件表格列名用"参数"而非"载荷" | 应为：事件/载荷/说明 |
| KanbanBoard | 插槽表格列名用"插槽名"和"参数" | 应为：插槽 / 作用域 / 说明 |
| Carousel | "可访问性"章节命名为"无障碍"，不统一 | 统一使用 `## 可访问性` |

## 四、组件文档增强建议

### 1. 可访问性章节（必须）

每个组件文档都应包含 `## 可访问性` 章节：

```markdown
## 可访问性

- **键盘操作**：支持 `Space` / `Enter` 触发，`Escape` 关闭
- **ARIA 属性**：自动管理 `aria-expanded`、`aria-controls` 等
- **焦点管理**：打开时焦点锁定在组件内，关闭时恢复焦点
```

### 2. 组合示例（推荐）

在复杂组件底部添加"相关组件"链接，引导用户发现更多组件。

### 3. 常见问题（推荐）

复杂组件（DataTable、Form、Toast）底部添加 FAQ 章节。
